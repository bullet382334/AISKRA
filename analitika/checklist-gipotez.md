# Чек-лист проверки гипотезы

> Используется, когда идея перешла в стадию «проверяется».

---

## Перед проверкой

- [ ] Гипотеза сформулирована чётко: «Мы предполагаем, что [X], потому что [Y]»
- [ ] Определено, что будет считаться подтверждением
- [ ] Определено, что будет считаться опровержением
- [ ] Назначен ответственный за проверку
- [ ] Установлен срок проверки

## Во время проверки

- [ ] Собраны факты (не мнения)
- [ ] Посмотрены аналоги / примеры
- [ ] Учтены ограничения (юридические, финансовые, временные)
- [ ] Проведён хотя бы один разговор с человеком, который уже это делал
- [ ] Посчитаны цифры (хотя бы грубо)

## После проверки

- [ ] Результат зафиксирован: подтверждена / частично / опровергнута
- [ ] Указан источник информации
- [ ] Обновлена карточка гипотезы
- [ ] Если подтверждена → определены следующие шаги
- [ ] Если опровергнута → зафиксировано, почему (чтобы не проверять снова)

---

## Пример заполнения

**Гипотеза:** Туристы готовы платить 5000 ₽/ночь за гостевой дом.

**Подтверждение:** Нашли 3+ аналога с такой ценой и заполняемостью 70%+.

**Опровержение:** Средний чек ниже 3000, заполняемость аналогов ниже 50%.

**Ответственный:** _[имя]_

**Срок:** _[дата]_

**Результат:** _[ожидает проверки]_
