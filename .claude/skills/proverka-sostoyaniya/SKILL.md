## Проверка состояния перед действиями

IMPORTANT: Перед любым действием с ботом, sync или внешними системами — проверить текущее состояние.

### Перед правкой bot.py
```bash
# Бот работает на СТАРОМ коде до перезапуска!
tasklist | grep python          # найти PID бота
wmic process where "ProcessId=PID" get CreationDate  # когда запущен
```
Если бот запущен ДО наших правок — предупредить: «Бот работает на старом коде, нужен перезапуск после правок».

### Перед запуском sync (только с разрешения!)
```bash
cat notion/.sync.lock           # есть ли lock?
cat notion/.sync_progress.json  # что делает текущий sync?
tasklist | grep python          # сколько python-процессов?
```
Если lock есть — проверить жив ли процесс. Если мёртв — stale lock, сообщить.

### После запуска sync
```bash
cat notion/.sync_progress.json  # прогресс
cat notion/.sync.lock           # PID
```
Проверить через 30-60 сек что progress обновляется.

### После завершения sync
```bash
cat notion/.sync_progress.json  # done: true?
cat notion/.sync.lock           # lock снят?
tasklist | grep python          # процесс завершился?
```
Если done=true но процесс висит и lock не снят — сообщить.

### Правило: не предполагать, проверять
- НЕ считать что бот перезапущен — проверить PID и время запуска
- НЕ считать что sync завершился — проверить progress и процесс
- НЕ считать что lock снят — проверить файл
- НЕ считать что правки применились — проверить что процесс запущен ПОСЛЕ правок
