## Что пользователь присылает и что делать

| Вход | Действие |
|------|----------|
| Транскрипция ответа на вопросы | Полный workflow (ниже) |
| Спонтанное голосовое | inbox/ → разбор → карточки → digest |
| Документ с вопросом | Анализ в контексте базы знаний |
| Вопрос по собранному | Ответ на основе digest + нужных файлов |
| «Что спросить дальше?» | Адаптивное интервью: `intervyu/workflow-adaptivnogo-intervyu.md` |
| Конкретная задача (расчёт, КП, бизнес-план) | ТОЛЬКО по прямому запросу пользователя → skill `workflow-realizaciya` → `realizaciya/` |

## Workflow обработки транскрипции

1. Сохранить → `inbox/ГГГГ-ММ-ДД-тема.md`
2. Разбор: ключевые мысли, эмоциональный тон, повторения, новое
3. **GATING:** Прочитать и применить правила размещения из `.claude/workflows/gating-rules.md`:
   - Извлечь сущности (люди, места, идеи, решения, задачи, факты)
   - Проверить каждую по `katalog.yaml`
   - Размещать по правилам: дополнить / создать / inbox / не затирать
4. Карточки → `karta-idej/{тип}/ГГГГ-ММ-ДД-название.md` (шаблоны: `shablony/`)
5. Обновить `karta-idej/zhurnal.md`
6. Если про главный проект → обновить `project/`
7. Если ответ на интервью → `intervyu/otvety/` + анализ → `intervyu/analiz/` + следующие вопросы
8. **Обновить каталог и digest:**
   - Новые сущности → добавить в `katalog.yaml`
   - Обновить нужный digest-файл (`digest-person.md` / `digest-proekty.md` / `digest-idei.md`)
   - Обновить `karta-idej/digest.md` (навигатор, ≤25 строк)
   - Обновить `_sostoyaniye.md` (≤40 строк)
9. Отчитаться пользователю: что извлечено, что обновлено, следующие вопросы
