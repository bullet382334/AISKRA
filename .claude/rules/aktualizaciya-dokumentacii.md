Источники правды: `_sostoyaniye.md` + `karta-idej/digest-proekty.md`.

| `project/sleduyushchie-shagi.md` | Список «Ждём и делаем» → секция «Сейчас» |

### 2. Очистка устаревшего

Для каждого файла — не только добавлять новое, но убирать мёртвое:

- `sleduyushchie-shagi.md`: выполненное → «Архив» (с датой и результатом). Убранное из _sostoyaniye → удалить из «Сейчас».
- `otkrytye-voprosy.md`: вопросы с ответами → «Выяснено» (дата + источник). Не оставлять [ ] если ответ уже есть в realizaciya/.
- `gipotezy.md`: подтверждённые/отклонённые → переместить из «Активных» в соответствующую секцию.
- `riski.md`: закрытые → пометить «ЗАКРЫТ» с датой. Неактуальные → удалить.
- `aktivy-i-finansy.md`: проданное → «продано (дата)». Одобренные кредиты → обновить статус.
- `uchastki.md`: отклонённые → пометить, сменить приоритет.

### 3. Проверка целостности

- **Битые ссылки:** grep по устаревшим путям. Заменить на актуальные файлы.
- **Несуществующие файлы:** ссылки вида `realizaciya/...md`, `karta-idej/idei/...md` — проверить, что файл существует.
- **Счётчики:** `digest.md` (идей, гипотез, исследований) — пересчитать по реальным файлам в папках.
- **Лимиты строк:** `_sostoyaniye.md` ≤ 40, `digest.md` ≤ 25, `digest-*.md` ≤ 40.
- **index.yaml:** новые файлы в `realizaciya/`, `karta-idej/idei/`, `inbox/` — должны быть в index.yaml.
- **katalog.yaml:** новые сущности (люди, места, решения) — должны быть в каталоге.
