## Безопасность секретов

- НИКОГДА не выводить токены, пароли, API-ключи в ответах пользователю
- НИКОГДА не копировать значения из .env в код или скрипты — использовать `os.environ`
- НИКОГДА не отправлять секреты в Telegram, логи или файлы в репозитории
- .env файлы редактируются ТОЛЬКО вручную пользователем
- При создании новых скриптов с секретами — добавлять переменные в .env.example (без реальных значений)
- Перед коммитом проверять: нет ли токенов в diff
