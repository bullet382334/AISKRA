## Правила размещения новой информации (gating rules)

IMPORTANT: Применять при КАЖДОЙ обработке транскрипций и создании новых файлов.
Цель: новая информация сразу попадает в правильное место, не теряется, не дублируется.

### Шаг 1. Извлечь сущности

Из новой информации выделить:
- **Люди** — имена, роли, кто упоминается
- **Места** — локации, участки, адреса
- **Идеи** — новые замыслы, форматы, продукты
- **Проекты** — конкретные инициативы с планом
- **Решения** — «мы решили», «я хочу», «будем делать»
- **Задачи** — «надо узнать», «проверить», «посчитать»
- **Факты/цифры** — цены, даты, размеры, юр. данные

### Шаг 2. Проверить по каталогу

Открыть `katalog.yaml` и для КАЖДОЙ извлечённой сущности:

| Ситуация | Действие |
|----------|----------|
| **Есть в каталоге, совпадает** | → Дополнить существующий файл. Не создавать новый |
| **Есть в каталоге, но новые детали** | → Дополнить файл + обновить `суть` в каталоге |
| **Нет в каталоге, понятно что это** | → Создать файл + добавить в `katalog.yaml` + обновить `index.yaml` |
| **Нет в каталоге, непонятно** | → Сохранить в `inbox/`, пометить для разбора |
| **Противоречит существующему** | → НЕ затирать. Добавить пометку: «Ранее: X. Теперь: Y (дата)» |
| **Дубликат** | → Не создавать. Ссылка на существующий файл |

### Шаг 3. Обновить связи

Если новая сущность связана с существующими — добавить в поле `связи` в `katalog.yaml` (в обе стороны).

### Шаг 4. Обновить digest

- Новый факт/решение → обновить соответствующий digest-файл:
  - Про заказчика → `karta-idej/digest-person.md`
  - Про проекты/факты/цифры → `karta-idej/digest-proekty.md`
  - Про идеи/мотивы/гипотезы → `karta-idej/digest-idei.md`
- Обновить главный `karta-idej/digest.md` если изменилась статистика
- Каждый digest-файл ≤ 40 строк

### Шаг 5. Обновить состояние

`_sostoyaniye.md` — только если изменился статус задач или появилась новая задача.

### Шаг 6. Чеклист после создания ЛЮБОГО файла

IMPORTANT: Ни один файл не должен существовать без записи в каталогах. Это ОБЯЗАТЕЛЬНЫЙ шаг.

При создании ЛЮБОГО нового .md файла — сразу:
1. **`katalog.yaml`** — добавить файл в поле `файлы` соответствующей сущности (или создать новую)
2. **`index.yaml`** — добавить строку с путём и статусом
3. **`realizaciya/index.md`** — если файл в `realizaciya/`, добавить строку в таблицу

Проверка: после завершения работы пройтись по всем созданным файлам и убедиться, что каждый есть в katalog.yaml и index.yaml. Если нет — это баг, исправить немедленно.

### Защита от ошибок

- **Никогда не затирать старые факты.** Цена была 9 млн, стала 7 млн? Записать оба.
- **Никогда не сливать разные сущности.** Два участка — два файла. Два человека с похожим именем — два файла.
- **Никогда не выдумывать связи.** Если связь не очевидна — не добавлять.
- **inbox/ — безопасная зона.** Непонятное → inbox. Лучше лишний раз положить в inbox, чем в неправильное место.
